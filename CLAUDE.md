# 项目通用开发规范

## 代码审查要点（每次修改后自动检查）

当用户要求 review 代码或完成代码修改后，请自动执行以下维度的检查：

1. **逻辑正确性**
   - 边界条件是否处理（空值、空列表、零值、超长字符串）
   - 循环/递归是否有退出条件
   - 异常捕获是否过宽（禁止 `except Exception: pass`）

2. **安全性**
   - 是否使用 SQL 参数化（禁止拼接）
   - 文件路径是否校验（防止路径遍历）
   - 外部 URL 是否验证（防止 SSRF）
   - 敏感信息是否硬编码（密钥、密码、token）

3. **健壮性**
   - 网络请求是否有超时和重试
   - 异步任务是否正确处理取消和异常
   - 资源是否有闭合（文件、数据库连接）

4. **Python 特定**
   - 类型注解是否完整
   - 是否使用 `logging` 而非 `print`
   - 是否遵循 PEP 8（行长度、命名）

5. **Vue 特定**
   - 组件 props 是否定义类型
   - 是否避免在模板中编写复杂逻辑
   - API 请求是否有错误处理
