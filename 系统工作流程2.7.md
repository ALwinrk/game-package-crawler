### 三级 HTTP 后端对比

| 后端 | 底层 | Cloudflare | 单页耗时 | 适用站点 |
|------|------|-----------|---------|---------|
| **Fetcher** | `Scrapling Fetcher` (curl_cffi + browserforge) | ❌ 被拦截 | 1-5s | Google Play / APKPure / APKCombo |
| **JS Render** | `StealthySession` (Chromium, CF=false) | ❌ 不等待 | 5-15s | APKPure 搜索页 JS 降级 |
| **StealthySession** | `StealthySession` (Chromium, CF=true) | ✅ 自动解 JS Challenge | 30-90s | APKMirror / APKVision |

## 二、排查模式

| 模式 | 路由 | 站点 | 后端 | 耗时 | 缓存 |
|------|------|------|------|------|------|
| **快速排查** (默认) | `/api/fetch` | Google Play + APKPure + APKCombo | Fetcher | ~12s | fast |
| **快速排查** | `/api/fetch/fast` | 同上 | Fetcher | ~12s | fast |
| **慢速排查** | `/api/fetch/slow` | APKMirror + APKVision | StealthySession | 30-90s | slow |
| **慢速异步** | `/api/fetch/slow/async` | APKMirror + APKVision | StealthySession | 30-90s (后台) | slow |
| **全量排查** | `/api/fetch/all` | 全部 5 个 | Fetcher + Stealthy | 30-100s | all |
| **多包名排查** | `/api/fetch/batch` | 同上, Semaphore 限流 | 对应模式 | ~12s/3包名 | 继承模式缓存 |

## 三、单包名快速排查流程 (v2.7: Google Play 双语言)

```
[用户] 输入 com.devsisters.ck
   │
   ▼
[前端] POST /api/fetch  {package, expected_version, expected_version_code, save_memo: true}
   │
   ▼
[Orchestrator] query_fast()
   │
   ├─ 检查 ScraperCache (TTL=300s) → 命中则直接返回
   ├─ Semaphore(scraper_concurrency=4) 限流
   ├─ asyncio.gather 并发 3 个快速爬虫:
   │
   ├─ GooglePlayScraper (v2.7: 双语言并行)
   │   ├─ asyncio.gather 并行请求:
   │   │   ├─ gp_app(pkg, lang="zh", country="cn")  → 标题 + 版本
   │   │   └─ gp_app(pkg, lang="en", country="us")  → recentChanges + 补充版本
   │   ├─ 版本: 中文站优先, 空/varies 则用英文站
   │   ├─ 标题: 中文站优先 (CookieRun: Kingdom)
   │   ├─ whats_new: 英文站优先, 中文站 fallback
   │   └─ Varies with device 时仍提取 app_name + whats_new
   │
   ├─ ApkpureScraper (v2.7: 搜索匹配加固 + 深度提取)
   │   ├─ Step 1: Fetcher 获取搜索页 → 包名路径验证
   │   │   └─ 包名必须在 URL 路径段中 (排除 search/推荐链接)
   │   ├─ Step 2: 详情页提取版本 + version_code
   │   │   └─ extract_both(html, package) + 包名关联限定
   │   └─ Step 3: 深度提取 whats_new
   │       └─ 匹配 .whats-new-container > p.text 段落
   │       └─ 跳过 "Last updated..." 行
   │       └─ html.unescape 解码实体 (v2.7)
   │
   └─ ApkcomboScraper
       ├─ Fetcher 获取 apkcombo.com/api/app/{pkg} → 302 详情页
       ├─ extract_both → version + version_code
       ├─ _extract_app_name (itemprop="name")
       └─ _extract_whats_new (data-role="whatsnew")
   │
   ▼
[结果聚合] _build_fetch_result()
   versions[] → 过滤 is_plausible_version() (v2.7: 拒绝>4段/单段>5位)
   best_version("7.5.102") — 多源共识
   best_version_code("205449") — APKPure 包名验证
   best_app_name("CookieRun: Kingdom") — 优先级 GP > APKPure > APKCombo
   best_whats_new(200+ chars) — 优先级 APKPure > APKCombo > GP (全部过滤占位符)
   │
   ▼
[写入缓存] ScraperCache.set(package, "fast", result) TTL=300s
   │
   ▼
[前端] ResultTable + 应用信息卡片
```

## 四、数据提取详解

### Google Play 双语言策略 (v2.7)

| 语言 | 用途 | 典型字段 |
|------|------|---------|
| `lang=zh, country=cn` | 标题、版本 | `title`(中文名), `version` |
| `lang=en, country=us` | 更新内容、版本补充 | `recentChanges`(完整 changelog), `version`(fallback) |

并行请求 (`asyncio.gather`)，总耗时 ≈ 单次请求。

### APKPure 更新内容提取 (v2.7)

```
HTML 结构:
  <div class="whats-new-module">
    <div class="title"><h2>What's New...</h2></div>
    <div class="whats-new-container">
      <p class="text update-date">Last updated on...</p>
      <div class="show-more"><div class="show-more-content">
        <p class="text">- New Feature 1</p>
        <p class="text">- New Feature 2</p>
      </div></div>
    </div>
  </div>

提取策略:
  1. 匹配 .whats-new-container 内所有 p.text
  2. 过滤 "Last updated" 行
  3. html.unescape 解码 (&amp; → &)
  4. 拼接为多行文本
```

### 版本号合理性校验 (v2.7)

```python
is_plausible_version("1.2.183")   → True
is_plausible_version("2.723.787") → True  (格式合法, 但可能被共识机制覆盖)
is_plausible_version("2723787")   → False (单段 >5 位)
is_plausible_version("1.2.3.4.5") → False (>4 段)
```

## 五、版本比对逻辑

```
两条线独立对比，互不覆盖:

版本号对比 (优先):
  compare_version_codes(expected_vc, best_vc)
    -1 → newer → "vc:21000 → vc:205449"
     0 → matched → "版本号一致 (vc:205449)"

版本名对比 (fallback):
  compare_versions(best_v, expected_v)
    1 → newer → "7.4.302 → 7.5.102"

综合状态 (compare_status):
  优先 version_code，其次 version_name

app_name 聚合: GP > APKPure > APKCombo
whats_new 聚合: APKPure > APKCombo > GP (全部过滤占位符)
```

## 八、批量 Excel 处理

```
上传 Excel → 解析列 → BatchTask → asyncio.create_task
  → Semaphore(5) + as_completed 并发
  → JSONL 流式写盘
  → 写回原 Excel: 排查时间 | 版本号(vc) | 对比状态
  → 完成弹窗: 已匹配/有新版本/版本较旧/未找到
  → BatchManager._cleanup_temp() 清理临时文件
```
