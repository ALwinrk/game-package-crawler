### 快/慢源对比

| | 快速源 (Fetcher) | 慢速源 (StealthySession) |
|---|---|---|
| **底层** | `curl_cffi` (模拟 Chrome TLS 指纹库) | `patchright` (Chromium 无头浏览器) |
| **原理** | 直接发 HTTP 请求 | 启动真实浏览器 → 加载页面 → JS 渲染 |
| **启动耗时** | 0s | 3-5s |
| **单页耗时** | 1-5s | 30-90s |
| **Cloudflare** | 被 403 拦截 | 自动解 JS Challenge |
| **适用站点** | Google Play / APKPure / APKCombo | APKMirror / APKVision |

## 二、排查模式

| 模式 | 路由 | 站点 | 耗时 | 说明 |
|------|------|------|------|------|
| **快速排查** (默认) | `/api/fetch` | Google Play + APKPure + APKCombo | ~10s | 日常使用 |
| **慢速排查** | `/api/fetch/slow` | APKMirror + APKVision | 30-90s | API 可用, 前端暂未接入 |
| **全量排查** | `/api/fetch/all` | 全部 5 个 | 30-100s | 需要完整覆盖时使用 |
| **多包名排查** | `/api/fetch/batch` | 同上, asyncio.gather 并发 | ~12s/3包名 | 文本框多行输入时自动触发 |

## 三、单包名排查流程

```
[用户] 输入 com.tencent.ig, 期望版本 vc:21000 / 4.3.0
   │
   ▼
[前端] POST /api/fetch  {package, expected_version, expected_version_code}
   │
   ▼
[Orchestrator] asyncio 并发 3 个快速爬虫
   │
   ├─ GooglePlayScraper
   │   ├─ google-play-scraper.app("com.tencent.ig", lang="en", country="us")
   │   ├─ 返回字段: version="4.4.0", updated=1777363707, size="1.3 GB"
   │   ├─ updated → 格式化为 "2026-04-28"
   │   └─ version_code: 不提供 (API 限制)
   │
   ├─ ApkpureScraper
   │   ├─ http_get("https://apkpure.com/search?q=com.tencent.ig")
   │   ├─ 提取详情页 URL (带包名验证)
   │   ├─ http_get(详情页) → extract_both(html, package="com.tencent.ig")
   │   │   关键修复 (v2.2):
   │   │   - 包名关联匹配: data-dt-package_name="target" 限定 500 字符内
   │   │   - 版本号位数: {1,12} 兼容短版本号 (如 89)
   │   │   - 避免推荐 APP 的数据污染
   │   ├─ 版本名: data-dt-version="4.4.0"
   │   └─ 版本号: data-dt-version_code="21118" (包名验证后)
   │
   └─ ApkcomboScraper
       ├─ http_get("https://apkcombo.com/api/app/com.tencent.ig")
       │   └─ 302 重定向到详情页
       ├─ extract_both(html, package="com.tencent.ig")
       ├─ 版本名: "4.4.0" (class="version" 或 schema.org itemprop)
       └─ 版本号: 视页面而定 (部分 APP 的页面不暴露)
   │
   ▼
[结果聚合]
   ├─ best_version("4.4.0") — 3源一致, 直接采纳
   ├─ best_version_code("21118") — 从 APKPure 获取 (包名验证)
   │
   ├─ 版本名对比:
   │   normalize("4.3.0") vs "4.4.0" → packaging.version.Version
   │   → "newer:4.3.0 → 4.4.0"
   │
   ├─ 版本号对比:
   │   int("21000") vs int("21118") → "newer:vc:21000 → vc:21118"
   │
   └─ compare_status: "newer" (优先取版本号对比结果)
   │
   ▼
[前端] ResultTable 渲染
   ┌────────────┬──────────┬──────────┬─────────────┬──────┐
   │ 数据源      │ 版本名    │ 版本号    │ 发布日期     │ 错误  │
   ├────────────┼──────────┼──────────┼─────────────┼──────┤
   │ Google Play │ 4.4.0    │ -        │ 2026-04-28  │ OK   │
   │ APKPure     │ 4.4.0    │ 21118    │ -           │ OK   │
   │ APKCombo    │ 4.4.0    │ -        │ -           │ OK   │
   └────────────┴──────────┴──────────┴─────────────┴──────┘

   版本名对比: newer:4.3.0 → 4.4.0
   版本号对比: newer:vc:21000 → vc:21118
```

## 四、版本比对逻辑

```
比对优先级: version_code (int 比较) > version_name (语义比较)

策略 1 — version_code 比对:
  int("21000") vs int("21118") → -1 → NEWER

策略 2 — version_name 比对 (fallback):
  packaging.version.Version("4.3.0") vs Version("4.4.0") → -1 → NEWER

策略 3 — normalize 兜底:
  normalize("v4.3.0") → "4.3.0"
  支持: 点分格式 (1.2.3), 字母后缀 (2.3.4-beta), Varies with device

综合状态: 同时输出两条比对线, 互不覆盖
  - version_name_compare: "newer:4.3.0 → 4.4.0"
  - version_code_compare: "newer:vc:21000 → vc:21118"
```

## 五、版本号解析修复 (v2.2 关键)

APKPure 详情页混杂了推荐 APP 的数据, 同一个 HTML 中同时存在:

```
主 APP (com.FataMorgana.Beauty2):
  data-dt-package_name="com.FataMorgana.Beauty2"
  data-dt-version_code="89"

推荐 APP (com.apkpure.aegon):
  data-dt-version_code="3206977"   ← 被误匹配!
```

修复方案 (`backend/core/parser.py`):

```python
# extract_version_code(html, package) — 新增 package 参数
# 当 package 提供时:
#   1. 搜索 data-dt-package_name="{target}" 
#   2. 限制 .{0,500}? 匹配范围 (同标签内, 不跨到推荐APP)
#   3. 提取该范围内的 data-dt-version_code
# 当 package 为空时:
#   保持原有全局匹配 (兼容测试和旧调用)
```

| 包名 | 修复前 version_code | 修复后 version_code |
|------|-------------------|-------------------|
| com.FataMorgana.Beauty2 | 3206977 (错误, 推荐APP的) | 89 (正确) |
| com.tencent.ig | 21118 (正确) | 21118 (不变) |
| com.tapblaze.coffeebusiness | 1001 (正确) | 1001 (不变) |

## 七、下载链接提取

```
[用户] 在结果表格点击 "获取链接"
   │
   ▼
[前端] POST /api/extract-links
  {source, detail_url, package}
   │
   ▼
[后端 extractors.py]

  ┌─ APKPure ──────────────────────────────────────┐
  │ 1. 提取 data-dt-apkid="b/APK/{base64}"        │
  │ 2. urlsafe base64 解码 → 验证含目标包名         │
  │ 3. 构造: https://d.apkpure.net/{apkid}         │
  │ 4. 无结果 → Playwright 网络拦截兜底              │
  └────────────────────────────────────────────────┘

  ┌─ APKCombo ─────────────────────────────────────┐
  │ 1. http_get 详情页 HTML                        │
  │ 2. 提取 /{slug}/{pkg}/download/{variant} 链接  │
  │ 3. 构造完整 URL                                 │
  │ 4. 无结果 → Playwright 网络拦截兜底              │
  └────────────────────────────────────────────────┘

  ┌─ APKMirror ────────────────────────────────────┐
  │ 1. stealth_get 搜索页 → 提取详情页链接          │
  │ 2. stealth_get 详情页 → 提取 /download/ 链接    │
  │ 3. stealth_get 下载页 → 提取 variant .apk 链接  │
  └────────────────────────────────────────────────┘
```

## 八、下载方式

| 源 | 直链下载 (aiohttp) | 浏览器下载 (window.open) | 原因 |
|----|-------------------|-------------------------|------|
| APKCombo | ❌ | ✅ | 下载页 URL 无防盗链, 浏览器正常打开即可 |
| APKPure | ❌ | ⚠️ | `d.apkpure.net` 强防盗链, 需真人解 captcha |
| APKMirror | ⚠️ | ✅ | CF 保护, 链接有时效性 |
| APKVision | ⚠️ | ✅ | 同上 |
| Google Play | ❌ | ❌ | 不提供 APK 下载 |

内置下载 (aiohttp + Range 断点续传) 仅对无防盗链直链有效, 403 时自动标记 error 并提示浏览器下载。

## 九、批量 Excel 处理

```
[用户] 上传 Excel (.xlsx/.xls)
  │ 必须包含: package_name 列
  │ 可选: expected_version_code, expected_version_name
  ▼
[前端] POST /api/batch/upload (multipart/form-data)
  │
  ▼
[后端] 解析 Excel → 构建结果映射 → asyncio.create_task 启动排查
  ← {"task_id": "batch_abc123", "total": 50, "filename": "games.xlsx"}
  │
  ▼
[前端] WebSocket 连接 ws://host/ws/batch_abc123
  │
  ▼
[后端 Batch Manager] 逐行排查 (快速模式, 3源)
  │ for each package_name:
  │   ├─ query_fast(package, expected_vc, expected_v)
  │   ├─ completed += 1
  │   └─ WebSocket 推送 {type: "batch_progress", progress_pct, status}
  │
  ▼ 100% 完成
[后端] 在原 Excel 中写入版本号列
  │
  ├─ 查找现有 "当前后台版本号(vc)" 列
  │   └─ 未找到则在包名列右侧插入新列
  ├─ 写入 best_version_code 到对应行
  └─ 未查到的写入 "-"
  │
  ▼
[前端] "下载结果 Excel" → 返回修改后的原始文件
```

### 结果 Excel 示例

```
处理前:
  package_name                   | app_name
  com.tapblaze.coffeebusiness    | Good Coffee
  com.FataMorgana.Beauty2        | Beauty2

处理后:
  package_name                   | 当前后台版本号(vc) | app_name
  com.tapblaze.coffeebusiness    | 1001               | Good Coffee
  com.FataMorgana.Beauty2        | 89                 | Beauty2
```

### 批量任务控制

```
暂停: POST /api/batch/{id}/pause  → 当前行完成后挂起 (asyncio.Event)
继续: POST /api/batch/{id}/resume → 从断点继续
取消: POST /api/batch/{id}/cancel → 终止并标记 cancelled
```

## 十、记忆化流程

```
[用户] 输入包名 → 点击 "应用上次版本"
   │
   ▼
[前端] GET /api/memo/{package_name}
   │
   ├─ 找到 → {found: true, data: {version_code, version_name, updated_at}}
   │   └─ 自动填入版本名/版本号输入框
   │
   └─ 未找到 → {found: false}
   │
[排查完成] save_memo=true
   │
   ▼
[后端] UPSERT user_history (package_name 为主键)
   └─ 仅保存最近一次输入
```

规则:
- ✅ 手动排查时保存 (`save_memo: true`)
- ❌ Excel 批量不触发 (避免污染)
- ❌ 多包名不触发 (仅单包时自动启用)
- 📦 存储: SQLite `user_history` 表

## 十六、关键设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 前后端分离 | FastAPI + Vue3, 单进程 | FastAPI 托管静态文件, 简化部署 |
| 快速/慢速源分离 | Fetcher vs StealthySession 两级 | 快源秒级优先, 慢源按需调用 |
| Playwright 后台启动 | asyncio.create_task | 不阻塞服务就绪 |
| 版本号包名验证 | 限定 500 字符 + {1,12} 位 | 避免推荐 APP 数据污染 |
| 下载防盗链处理 | 提取链接 → 用户浏览器打开 | APK CDN 强防盗链, 无头浏览器无法突破 |
| 批量写回原表 | 在原 Excel 插入版本号列 | 与旧版 fill_version_codes.py 行为一致 |
| 批量任务调度 | asyncio.create_task | 比 BackgroundTasks 更可靠 |
| 断点续传 | aiohttp + Range 头 + .part 文件 | 网络中断可恢复下载 |
| 选择器外置 | selectors.yaml | 站点改版无需重编译 |
| 全局浏览器 | Playwright 单例 + Semaphore(2) | 实例复用避免过载 |
| 版本比对 | packaging.version + int 比较 | 支持语义版本和纯数字 |
| 多包名并发 | asyncio.gather 并发查询 | 3 包名 ~12s vs 串行 30s |
| 日志格式 | loguru {} 风格 | 替换遗留 %s/%d (v2.2 全量修复) |
| EXE 打包 | PyInstaller 单文件 | 177MB, ~40s 启动 |

## 十七、站点数据解析要点

| 站点 | 版本名来源 | 版本号来源 | 发布时间 | 下载链接 |
|------|-----------|-----------|---------|---------|
| Google Play | `app().version` | ❌ API 不提供 | `updated` → YYYY-MM-DD | ❌ 不提供 |
| APKPure | `data-dt-version` | `data-dt-version_code` (包名验证, 500字符限定) | — | `d.apkpure.net/{apkid}` |

| APKMirror | `class="version"` | 页面文本 | — | 多步跳转提取 |
| APKVision | `.download-link` | 页面文本 | — | `.download-link` |
