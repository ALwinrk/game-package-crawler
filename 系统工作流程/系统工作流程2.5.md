### 三级 HTTP 后端对比

| 后端 | 底层 | Cloudflare | 单页耗时 | 适用站点 |
|------|------|-----------|---------|---------|
| **Fetcher** | `Scrapling Fetcher` (curl_cffi + browserforge) | ❌ 被拦截 | 1-5s | Google Play / APKPure / APKCombo |
| **JS Render** | `StealthySession` (Chromium, CF=false) | ❌ 不等待 | 5-15s | APKPure 搜索页 JS 降级 |
| **StealthySession** | `StealthySession` (Chromium, CF=true) | ✅ 自动解 JS Challenge | 30-90s | APKMirror / APKVision |

## 二、排查模式

| 模式 | 路由 | 站点 | 后端 | 耗时 | 缓存 |
|------|------|------|------|------|------|
| **快速排查** (默认) | `/api/fetch` | Google Play + APKPure + APKCombo | Fetcher | ~10s | fast |
| **快速排查** | `/api/fetch/fast` | 同上 | Fetcher | ~10s | fast |
| **慢速排查** | `/api/fetch/slow` | APKMirror + APKVision | StealthySession | 30-90s | slow |
| **慢速异步** | `/api/fetch/slow/async` | APKMirror + APKVision | StealthySession | 30-90s (后台) | slow |
| **全量排查** | `/api/fetch/all` | 全部 5 个 | Fetcher + Stealthy | 30-100s | all |
| **多包名排查** | `/api/fetch/batch` | 同上, Semaphore 限流 | 对应模式 | ~12s/3包名 | 继承模式缓存 |

## 三、单包名快速排查流程

```
[用户] 输入 com.tencent.ig
   │
   ▼
[前端] POST /api/fetch  {package, expected_version, expected_version_code, save_memo: true}
   │
   ▼
[Orchestrator] query_fast()
   │
   ├─ 检查 ScraperCache (TTL=300s) → 命中则直接返回
   │
   ├─ Semaphore(scraper_concurrency=4) 限流
   │
   ├─ asyncio.gather 并发 3 个快速爬虫:
   │
   ├─ GooglePlayScraper
   │   ├─ 注入代理到 urllib 环境变量
   │   ├─ google-play-scraper.app(package, lang="en", country="us")
   │   ├─ 可加载 Cookie 文件 (google_play_cookie_path)
   │   └─ version="4.4.0", updated→"2026-04-28", size="1.2 GB"
   │
   ├─ ApkpureScraper
   │   ├─ Step 1: Fetcher 获取搜索页 apkpure.com/search?q={pkg}
   │   │   └─ 若被 CF 拦截 → 回退到 apkpure.net
   │   ├─ Step 2: 正则提取详情页 URL (含包名匹配, 排除 /search)
   │   │   └─ 若 Fetcher 无链接 → JS Render 降级取详情
   │   ├─ Step 3: Fetcher 获取详情页
   │   └─ Step 4: extract_both(html, package) → version + version_code
   │       └─ 包名关联: data-dt-package_name + .{0,500}? 限定
   │
   └─ ApkcomboScraper
       ├─ Fetcher 获取 apkcombo.com/api/app/{pkg}
       ├─ 服务端 302 重定向到详情页
       └─ extract_both(html, package) → version + version_code
   │
   ▼
[结果聚合] _build_fetch_result()
   best_version("4.4.0") — 多源共识判定 (优先Google Play)
   best_version_code("21118") — APKPure 包名验证获取
   version_name_compare: "matched:4.3.0 → 4.4.0" (NEWER)
   version_code_compare: "matched:vc:21000 → vc:21118" (NEWER)
   │
   ▼
[写入缓存] ScraperCache.set(package, "fast", result) TTL=300s
   │
   ▼
[记忆化] save_memo=true → MemoStore.upsert()
   │
   ▼
[前端] ResultTable 展示
```

## 四、慢速异步排查流程

```
[前端] POST /api/fetch/slow/async  {package, expected_version, expected_version_code}
   │
   ▼
[后端] SlowTaskStore.create() → task_id = "slow_xxxxxxxx"
   ← {"task_id": "slow_xxx", "status": "pending", "message": "任务已提交，预计 1-2 分钟完成"}
   │
   ▼
[后台] asyncio.create_task(_run_slow())
   │
   ├─ query_slow(package) → APKMirrorScraper + ApkvisionScraper
   │   ├─ 检查 ScraperCache("slow") → 命中则跳过
   │   └─ StealthySession 每站点 30-90s
   │
   ├─ 成功 → SlowTaskStore.complete(task_id, result)
   │   └─ WebSocket 推送: {type: "slow_task_completed", data: {task_id, status}}
   │
   └─ 失败 → SlowTaskStore.fail(task_id, error)
       └─ WebSocket 推送: {type: "slow_task_error", data: {task_id, status, error}}
   │
   ▼
[前端] 轮询 GET /api/fetch/slow/result/{task_id}
   └─ pending → 继续轮询
   └─ completed → 展示结果
   └─ error → 显示错误
```

## 五、版本比对逻辑

```
两条线独立对比，互不覆盖:

版本号对比 (优先):
  compare_version_codes(expected_vc, best_vc)
    -1 → newer → "vc:21000 → vc:21118"
     0 → matched → "版本号一致 (vc:21118)"
     1 → older → "vc:21000 → vc:20000"
  None → 无法解析，回退到版本名对比

版本名对比 (fallback):
  compare_versions(best_v, expected_v)
    1 → newer → "4.3.0 → 4.4.0"
    0 → matched (含 normalize 后一致)
    -1 → older → "4.5.0 → 4.4.0"

综合状态 (compare_status):
  优先 version_code 对比结果，其次 version_name 对比结果
  均无 → NOT_FOUND
```

### best_version / best_version_code 判定

| 条件 | 策略 |
|------|------|
| 同一值 ≥2 源一致 | 直接采纳 |
| 仅 1 源有结果 | 采纳该源 |
| 多源有结果无共识 | 优先 Google Play，否则取数值最大 |

| 优先级 | 模式 | 场景 | 关键修复 |
|--------|------|------|----------|
| 0a | `data-dt-package_name` + `.{0,500}?` + `data-dt-version_code` | APKPure 详情页 | **包名关联 + 500字符限定** |
| 0b | `data-version_code="NNNNNN"` | 带下划线变体 | |
| 1 | `variant code: NNNNNN` | APKCombo | |
| 2 | `x.y.z (NNNNNN)` | 括号内嵌 | |
| 3a | `data-dt-versioncode="NNNNNN"` | APKPure 搜索页 | |
| 3b | `data-versioncode="NNNNNN"` | | |
| 4 | `<meta>` versionCode | HTML meta | |
| 5 | `"versionCode": "NNNNNN"` | JSON 内嵌 | |
| 6 | `Version Code: NNNNNN` | 纯文本标签 | |
| 7 | `data-app-versioncode` | 备选 data 属性 | |
| 8 | `_NNNNNN_.apk` / `-NNNNNN.apk` | APK 文件名 | |
| 9 | `<dt>Version Code</dt><dd>NNNNNN</dd>` | 定义列表 | |
| 10 | `version/ver/vc/code : NNNNN` | 兜底 | |

### 版本名解析 (5 策略)

1. class 名含 "version" 的元素 → 精确匹配 x.y.z
2. data-* 属性 (data-dt-version, data-version 等)
3. Schema.org itemprop="version"
4. 全文 "Version: x.y.z" / "版本：x.y.z"
5. 兜底 `>x.y.z<` 标签内嵌

## 七、TTL 缓存系统

```
ScraperCache
├── 内存存储: { "pkg::mode" → (FetchResult, expiry_time) }
├── TTL 可配置: cache_ttl_seconds (默认 300s = 5分钟)
├── 粒度: 按 package + mode (fast/slow/all) 分别缓存
├── API: GET (命中返回), SET (写入), CLEAR (清除全部)
│
├── 命中流程:
│     query_fast("com.tencent.ig") → cache.get("com.tencent.ig", "fast")
│     → 命中 → 直接返回 FetchResult (跳过全部 HTTP 请求)
│     → 未命中 → 执行爬虫 → cache.set()
│
└── 管理端点: POST /api/cache/clear → 清除所有缓存
```

### SlowTaskStore (慢速异步任务存储)

```
SlowTaskStore
├── 内存存储: { "slow_xxxxxxxx" → {status, result, error, created_at} }
├── TTL: 3600s (1小时)
├── 自动清理过期任务
│
├── create()     → 生成 task_id, 状态 pending
├── complete(id) → 写入结果, 状态 completed
├── fail(id)     → 写入错误, 状态 error
└── get(id)      → 返回任务状态 (自动清理过期)
```

## 八、多包名并发排查

```
前端:
  多行/逗号/分号分隔 → packages.length > 1
  → POST /api/fetch/batch  {packages: [...], mode: "fast"}
  → 12s 返回全部结果

后端 query_batch():
  Semaphore(batch_concurrency=5) 限流 (可配置)
  asyncio.gather 全部协程同时启动, Semaphore 控制并发度
  每个包名内部走对应的 query_fn (继承各自的 ScraperCache)
  返回: [FetchResult, ...] 过滤 None (停止/取消时)
```

## 九、批量 Excel 处理

### 完整流程

```
[用户] 上传 Excel (.xlsx/.xls)
  │ 必须: package_name 列
  │ 可选: expected_version_code, expected_version_name
  ▼
[后端] 解析 Excel → 创建 BatchTask → asyncio.create_task 启动
  │ 自动检测列: package_name/package/packagename,
  │             version_code/versioncode,
  │             version_name/versionname
  ← {"task_id": "batch_xxx", "total": N, "filename": "xxx.xlsx"}
  │
  ▼
[前端] WebSocket ws://host/ws/{task_id} 订阅进度
  │
  ▼
[后端] Semaphore(batch_concurrency=5) + as_completed 并发
  │ 每个包名: query_fast(pkg) → best_version_code + compare_status
  │ 进度推送: 每 5 个完成推送一次 (控制频率)
  │ 流式持久化: 结果写入临时 JSONL 文件 (降低内存)
  │
  ▼ 100%
[后端] 在原 Excel 写入结果 (保留原始样式):
  │ 自动检测/新建三列:
  │   排查时间              → 2026-06-04 10:30
  │   当前后台版本号(vc)    → 21118 / 1001 / -
  │   对比状态              → 已匹配 / 有新版本 / 版本较旧 / 未找到 / 错误
  │   列样式: 蓝色表头 + 白色粗体 + 居中
  │
  ▼
[前端] 弹窗显示汇总:
  ┌─────────────────────────┐
  │ 排查完成! 共 N 个包名    │
  │ 已匹配:   X (绿色)      │
  │ 有新版本: Y (橙色)      │
  │ 版本较旧: Z (灰色)      │
  │ 未找到:   W (红色)      │
  │                          │
  │ 点击下载结果 Excel        │
  └─────────────────────────┘
  │
  ▼
[用户] 点击 "下载结果 Excel" → GET /api/batch/{id}/download
  → BatchManager.export_to_excel() 写回结果
  → BatchManager._cleanup_temp() 清理临时 JSONL 文件
  → StreamingResponse 返回修改后的 Excel
```

### 批量内部并发架构

```
BatchManager.run(task):
  Semaphore(batch_concurrency=5)
  ↓
  as_completed([_process_one(0), _process_one(1), ..., _process_one(N)])
  ↓
  每个 _process_one 内部:
    async with semaphore:          ← 最多 5 个同时执行
      检查取消/暂停状态
      query_fast(pkg, ev, evc)     ← 每个包名内部还有 scraper_concurrency=4
      返回 (idx, FetchResult)
  ↓
  按原始索引排序 → task.results
  ↓
  _flush_results_to_temp(task)     ← 流式写入 JSONL 临时文件
```

### 控制

```
暂停: POST /api/batch/{id}/pause  → BatchTask.status = "paused", stop_event.set()
继续: POST /api/batch/{id}/resume → BatchTask.status = "running", stop_event.clear()
取消: POST /api/batch/{id}/cancel → BatchTask.status = "cancelled", BatchManager._cleanup_temp()
```

## 十、下载系统

### 下载链接提取

| 源 | 方法 | 链接格式 |
|----|------|---------|
| APKPure | URL 构造 → 浏览器下载页 | `apkpure.net/cn/{slug}/{pkg}/download` |
| APKCombo | URL 构造 → 浏览器下载页 (中文) | `apkcombo.com/zh/{slug}/{pkg}/download/phone-{ver}-apk` |
| APKMirror | StealthySession 多步跳转 → 正则提取 variant .apk | 动态 |
| APKVision | StealthySession 页面提取 → 正则 .apk 链接 | 动态 |
| Google Play | ❌ | 不提供 |

### 下载后端

| 源 | aiohttp 直下 | Playwright 浏览器下载 | 原因 |
|----|-------------|---------------------|------|
| APKCombo | ❌ | ✅ 浏览器下载页 | 防盗链 |
| APKPure | ❌ | ✅ 浏览器下载页 | d.apkpure.net 强防盗链 |
| APKMirror | ⚠️ | ✅ | CF 保护, 链接有时效 |
| APKVision | ⚠️ | ✅ | 同上 |

### Playwright 浏览器下载流程 (download/manager.py)

```
aiohttp 返回 403/404
  ↓
_browser_download(task, part_path):
  ├─ 启动独立 patchright 浏览器 (sync_playwright)
  ├─ APKPure: 先访问 detail_url 建立 Cookie 上下文
  ├─ 访问 download URL → 监听 download 事件
  ├─ 若未触发: 点击页面下载按钮 (5 种选择器尝试)
  ├─ 等待 8s → 检查文件大小 ≥ 50KB
  ├─ 移动到目标路径
  └─ 超时 120s → error
```

### aiohttp 断点续传

```
DownloadManager._download():
  检查 .part 文件 → 读取已下载字节
  → Range: bytes={downloaded_size}- 请求
  → aiohttp 流式写入 (chunk_size 可配, 默认 1MB)
  → 每 0.5s 计算速度/进度 → WebSocket 推送
  → 完成: 重命名 .part → .apk
```
