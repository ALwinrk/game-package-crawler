## 二、排查模式

| 模式 | 路由 | 站点 | 耗时 |
|------|------|------|------|
| **快速排查** (默认) | `/api/fetch` | Google Play + APKPure + APKCombo | ~10s |
| **慢速排查** (API可用,前端未接) | `/api/fetch/slow` | APKMirror + APKVision | 30-90s |
| **全量排查** | `/api/fetch/all` | 全部 5 个 | 30-100s |

## 三、单包名排查流程

```
用户输入包名 com.tencent.ig
  │
  ▼
[前端] POST /api/fetch  {package: "com.tencent.ig"}
  │
  ▼
[后端 Orchestrator] 并发 3 个快速爬虫
  │
  ├─ GooglePlayScraper
  │   ├─ google-play-scraper.app("com.tencent.ig")
  │   ├─ 返回: version="4.4.0", updated=Unix时间戳
  │   └─ Unix → "2026-04-28" 格式化
  │
  ├─ ApkpureScraper
  │   ├─ http_get("https://apkpure.com/search?q=com.tencent.ig")
  │   ├─ 提取详情页 URL: /pubg-mobile-.../com.tencent.ig
  │   ├─ http_get(详情页) → extract_both(html)
  │   ├─ 版本名: 4.4.0  (data-dt-version)
  │   └─ 版本号: 21118   (data-dt-version_code, 优先于 data-dt-versioncode)
  │
  └─ ApkcomboScraper
      ├─ http_get("https://apkcombo.com/api/app/com.tencent.ig")
      │   └─ 302 重定向到详情页
      ├─ extract_both(html)
      ├─ 版本名: 4.4.0  (class="version")
      └─ 版本号: 21118   (variant code: NNNNNN) 或 无 (站点不暴露)
  │
  ▼
[结果聚合]
  ├─ best_version = 4.4.0 (2/3 源一致, 达成共识)
  ├─ best_version_code = "21118"
  ├─ version_name_compare: "matched:4.4.0"
  ├─ version_code_compare: "matched:vc:21118"
  └─ compare_status: matched
  │
  ▼ 返回 JSON
[前端] ResultTable 渲染表格
  ┌────────────┬──────────┬──────────┬─────────────┐
  │ 数据源      │ 版本名    │ 版本号    │ 发布日期     │
  ├────────────┼──────────┼──────────┼─────────────┤
  │ Google Play │ 4.4.0    │ -        │ 2026-04-28  │
  │ APKPure     │ 4.4.0    │ 21118    │ -           │
  │ APKCombo    │ 4.4.0    │ -        │ -           │
  └────────────┴──────────┴──────────┴─────────────┘
```

## 四、版本比对逻辑

```
用户输入期望版本: version_name="4.3.0", version_code="21000"
线上最佳版本:     best_version="4.4.0",  best_version_code="21118"

比对 (独立两条线):
  ┌─ 版本名比对 ─────────────────────┐
  │ normalize("4.3.0") vs "4.4.0"   │
  │ → packaging.version.Version 比较 │
  │ → 4.3.0 < 4.4.0 → NEWER        │
  │ → 输出: "newer:4.3.0 → 4.4.0"  │
  └─────────────────────────────────┘

  ┌─ 版本号比对 ─────────────────────┐
  │ int("21000") vs int("21118")    │
  │ → 21000 < 21118 → NEWER        │
  │ → 输出: "newer:vc:21000 → vc:21118" │
  └─────────────────────────────────┘

综合状态: NEWER (优先取版本号比对结果)
```

比对策略优先级:
1. **version_code 比对**: 若双方都有 → int 比较 (最精确)
2. **version_name 比对**: 无 version_code 时 → `packaging.version.Version` 语义比较
3. 支持点分格式 (1.2.3)、字母后缀 (2.3.4-beta)

## 五、下载链接提取

```
用户点击 "获取链接" 按钮
  │
  ▼
[前端] POST /api/extract-links  {source, detail_url, package}
  │
  ▼
[后端 extractors.py]

  ┌─ APKPure ──────────────────────────────────────┐
  │ 1. 提取 data-dt-apkid="b/APK/{base64}"        │
  │ 2. urlsafe base64 解码 → 验证包名              │
  │ 3. 构造: https://d.apkpure.net/{apkid}         │
  │ 4. 失败 → Playwright 网络拦截兜底              │
  └────────────────────────────────────────────────┘

  ┌─ APKCombo ─────────────────────────────────────┐
  │ 1. 提取 /{slug}/{pkg}/download/{variant} 链接  │
  │ 2. 构造完整 URL: https://apkcombo.com/...     │
  │ 3. 失败 → Playwright 网络拦截兜底              │
  └────────────────────────────────────────────────┘

  ┌─ APKMirror ────────────────────────────────────┐
  │ 1. StealthySession 搜索页 → 提取详情页         │
  │ 2. 详情页 → 提取 /apk/.../download/ 链接       │
  │ 3. 下载页 → 提取 variant .apk 链接             │
  └────────────────────────────────────────────────┘

  ┌─ APKVision ────────────────────────────────────┐
  │ 1. StealthySession 搜索页                      │
  │ 2. 提取页面中 .apk 下载链接                     │
  └────────────────────────────────────────────────┘
```

## 六、下载管理器

```
用户点击 "浏览器下载" 按钮
  │
  ▼
[前端] 用 window.open() 在浏览器打开下载 URL
  │ (APKCombo: 页面无防盗链, 直接触发下载)
  │ (APKPure: 需用户浏览器解 captcha 后下载)
  │
  ▼
[内置下载] POST /api/download
  │
  ├─ aiohttp 流式下载 (64KB chunks)
  ├─ Range 断点续传 (.part 文件)
  ├─ 403 时回退到 Playwright 浏览器尝试
  └─ WebSocket 推送进度 (百分比 + 速度)
```

下载状态:

| 源 | 直链下载 | 浏览器下载 | 原因 |
|----|---------|-----------|------|
| APKCombo | ❌ | ✅ /download/ 页面 | 无防盗链 |
| APKPure | ❌ | ⚠️ 需解 captcha | `d.apkpure.net` 防盗链 |
| APKMirror | ⚠️ | 可用 | CF 保护不稳定 |
| APKVision | ⚠️ | 可用 | 同上 |
| Google Play | ❌ | ❌ | 不提供 APK 下载 |

## 七、批量 Excel 处理

```
用户上传 Excel (.xlsx/.xls)
  │ 必须包含: package_name 列
  │ 可选: expected_version_code, expected_version_name
  ▼
[前端] POST /api/batch/upload  (multipart/form-data)
  │
  ▼
[后端] 解析 Excel → 创建 BatchTask → 生成 task_id
  │ 返回 {"task_id": "batch_abc123", "total": 50}
  ▼
[前端] WebSocket 连接 ws://host/ws/batch_abc123
  │
  ▼
[后端 Batch Manager] 后台协程逐行处理
  │ for each row:
  │   ├─ query_single(package, expected_v, expected_vc)
  │   ├─ completed += 1
  │   └─ WebSocket 推送 {type: "batch_progress", progress_pct, status}
  │
  ▼ 完成后
[前端] 进度条 100% → "下载结果 Excel" 按钮
  │
  ▼
[后端] GET /api/batch/{task_id}/download → Excel 流式返回
```

批量任务控制:
- **暂停**: POST /api/batch/{id}/pause → asyncio.Event.set()
- **继续**: POST /api/batch/{id}/resume → asyncio.Event.clear()
- **取消**: POST /api/batch/{id}/cancel

## 八、记忆化流程

```
[用户] 输入包名 com.tencent.ig
  │
  ▼
[前端] GET /api/memo/com.tencent.ig
  │
  ├─ 找到 → {"found": true, "data": {version_code: "21118", version_name: "4.4.0"}}
  │   └─ 用户点击 "应用上次版本" → 自动填充输入框
  │
  └─ 未找到 → {"found": false}
  │
  ▼
[排查完成] save_memo=true 时
  │
  ▼
[后端] POST /api/memo → SQLite UPSERT
  │   package_name="com.tencent.ig"
  │   version_code="21118"
  │   version_name="4.4.0"
  │
  ▼
[设置页] 可查看/管理所有记忆数据
```

记忆规则:
- ✅ 手动排查时保存 (勾选"保存本次版本")
- ❌ Excel 批量不触发 (避免污染)
- 📦 存储: SQLite `user_history` 表 (以 package_name 为主键)

## 十二、关键设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 前后端分离 | FastAPI + Vue3, 单进程 | 简化部署, FastAPI 托管静态文件 |
| 快速/慢速源分离 | 快源 (Fetcher) + 慢源 (StealthySession) | 快源秒级, 慢源需浏览器 |
| 下载防盗链 | 提取链接 → 用户浏览器打开 | APK 站点防盗链极强, 无头浏览器无法突破 |
| 批量任务 | asyncio.Queue + BackgroundTasks | 轻量, 无需 Redis/Celery |
| 断点续传 | aiohttp + Range 头 + .part 文件 | 网络中断可恢复 |
| 选择器外置 | selectors.yaml | 站点改版无需重编译 |
| 全局浏览器 | Playwright 单例 + Semaphore(2) | 实例复用, 避免过载 |
| 版本比对 | packaging.version + int 比较 | 支持语义版本和纯数字 |
