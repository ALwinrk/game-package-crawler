# 游戏包名爬虫系统 — 实施计划

> 基于《游戏包名爬虫系统设计文档（最终版）》，复用现有 game-version-check 工具代码
> 项目路径：`C:\new_version`

---

## 一、架构概览

```
[Electron + Vue3 + Tailwind CSS + Element Plus]  前端
        │ WebSocket + REST API
[FastAPI]                                          后端
        │
[Orchestrator] ── 爬取调度器
  ├─ GooglePlayScraper   (google-play-scraper + Playwright降级)
  ├─ ApkpureScraper      (绕过 Cloudflare)
  ├─ ApkcomboScraper     (页面 JSON 解析)
  ├─ ApkmirrorScraper    (二次请求下载页)
  └─ ApkvisionScraper    (静态页面解析)
        │
[Result Aggregator + Version Comparator]
        │
[Download Manager] ── aiohttp 异步下载 + Range 断点续传
[Memoization Store] ── SQLite user_history
[Batch Task Manager] ── asyncio.Queue + BackgroundTasks
```

### 关键设计决策
- ❌ 移除 FDM 外部下载器 → aiohttp 内置异步下载 + 断点续传
- ❌ 移除 Celery/Redis → FastAPI BackgroundTasks + asyncio.Queue
- ✅ 保留 Google Play 爬虫，提供 Cookie 导入机制
- ✅ 架构选择仅在站点提供多个 ABI 时显示

## 四、核心工作流

### 1. 单包名查询
```
用户输入包名 → POST /api/fetch → orchestrator.query()
  → asyncio 并发 5 个爬虫 → 版本比较 → 返回结果 + 对比状态
```

### 2. 批量 Excel
```
上传 Excel → POST /api/batch/upload → 生成 task_id
  → BackgroundTasks 启动处理 → WebSocket 推送进度
  → 完成后返回结果 Excel
```

### 3. 异步下载 + 断点续传
```
勾选版本 → POST /api/download → 加入 asyncio.Queue
  → aiohttp 流式下载(64KB chunk) → Range 断点续传
  → WebSocket 推送进度(百分比+速度)
```

### 4. 记忆化输入
```
输入包名(防抖300ms) → GET /api/memo/{pkg} → SQLite 查询
  → 自动填充版本号字段
```
