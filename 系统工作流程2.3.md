### 快/慢源对比

| | 快速源 (Fetcher) | 慢速源 (StealthySession) |
|---|---|---|
| **底层** | `curl_cffi` (模拟 Chrome TLS 指纹) | `patchright` (Chromium 无头浏览器) |
| **原理** | 直接发 HTTP 请求 | 启动浏览器 → 加载页面 → 执行 JS |
| **单页耗时** | 1-5s | 30-90s |
| **Cloudflare** | 被 403 拦截 | 自动解 JS Challenge |
| **适用站点** | Google Play / APKPure / APKCombo | APKMirror / APKVision |

## 二、排查模式

| 模式 | 路由 | 站点 | 耗时 |
|------|------|------|------|
| **快速排查** (默认) | `/api/fetch` | Google Play + APKPure + APKCombo | ~10s |
| **慢速排查** | `/api/fetch/slow` | APKMirror + APKVision | 30-90s (API可用, 前端未接) |
| **全量排查** | `/api/fetch/all` | 全部 5 个 | 30-100s |
| **多包名排查** | `/api/fetch/batch` | 同上, asyncio.gather 并发 | ~12s/3包名 |

## 三、单包名排查流程

```
[用户] 输入 com.tencent.ig
   │
   ▼
[前端] POST /api/fetch  {package, expected_version, expected_version_code}
   │
   ▼
[Orchestrator] asyncio 并发 3 个快速爬虫
   │
   ├─ GooglePlayScraper
   │   └─ google-play-scraper.app() → version="4.4.0", updated→"2026-04-28"
   │
   ├─ ApkpureScraper
   │   ├─ 搜索页 http_get → 提取详情页 URL
   │   ├─ 详情页 http_get → extract_both(html, package)
   │   │   └─ 包名关联: data-dt-package_name + 500字符限定
   │   └─ version="4.4.0", version_code="21118"
   │
   └─ ApkcomboScraper
       ├─ /api/app/{pkg} → 302 详情页
       └─ extract_both(html, package) → version="4.4.0"
   │
   ▼
[结果聚合]
   best_version("4.4.0") — 3源一致
   best_version_code("21118") — APKPure 获取 (包名验证)
   version_name_compare: "4.3.0 → 4.4.0" (NEWER)
   version_code_compare: "vc:21000 → vc:21118" (NEWER)
   │
   ▼
[前端] ResultTable
   ┌────────────┬──────────┬──────────┬─────────────┐
   │ 数据源      │ 版本名    │ 版本号    │ 发布日期     │
   ├────────────┼──────────┼──────────┼─────────────┤
   │ Google Play │ 4.4.0    │ -        │ 2026-04-28  │
   │ APKPure     │ 4.4.0    │ 21118    │ -           │
   │ APKCombo    │ 4.4.0    │ -        │ -           │
   └────────────┴──────────┴──────────┴─────────────┘
```

## 四、版本比对逻辑

```
两条线独立对比，互不覆盖:

版本号对比 (优先):
  int("21000") vs int("21118") → newer → "vc:21000 → vc:21118"

版本名对比 (fallback):
  packaging.version.Version("4.3.0") vs "4.4.0") → newer → "4.3.0 → 4.4.0"

支持:
  - 点分格式 (1.2.3)
  - 字母后缀 (2.3.4-beta)
  - Varies with device → 视为空
```

## 五、版本号解析修复 (v2.2 关键)

APKPure 详情页混杂推荐 APP，同一 HTML 中存在多个 `data-dt-version_code`。

```
主 APP: data-dt-package_name="com.FataMorgana.Beauty2" data-dt-version_code="89"
推荐:   data-dt-version_code="3206977"  ← 被误匹配
```

修复 (`backend/core/parser.py`):

| 修复点 | 变更 | 说明 |
|--------|------|------|
| 包名关联 | `extract_version_code(html, package)` | 新增 package 参数 |
| 范围限定 | `.*?` → `.{0,500}?` | 限定同标签内, 不跨到推荐APP |
| 位数兼容 | `{3,12}` → `{1,12}` | 兼容短版本号 (如 89) |

## 九、批量 Excel 处理

### 完整流程

```
[用户] 上传 Excel (.xlsx/.xls)
  │ 必须: package_name 列
  │ 可选: expected_version_code, expected_version_name
  ▼
[后端] 解析 Excel → 创建 BatchTask → asyncio.create_task 启动
  ← {"task_id": "batch_xxx", "total": N}
  │
  ▼
[前端] WebSocket ws://host/ws/{task_id} 订阅进度
  │
  ▼
[后端] 逐行排查 (快速模式, 3源)
  │ for each:
  │   query_fast(pkg) → best_version_code + compare_status
  │   WebSocket 推送 {progress_pct, status}
  │
  ▼ 100%
[后端] 在原 Excel 写入结果:
  │ 新增三列 (自动检测已有列, 无则新建):
  │   排查时间         → 2026-06-03 18:02
  │   当前后台版本号(vc) → 21118 / 1001 / -
  │   对比状态          → 已匹配 / 有新版本 / 版本较旧 / 未找到
  │
  ▼
[前端] 弹窗显示汇总:
  ┌─────────────────────┐
  │ 排查完成! 共 N 个包名  │
  │ 已匹配:   X          │
  │ 有新版本: Y          │
  │ 版本较旧: Z          │
  │ 未找到:   W          │
  │                      │
  │ 点击下载结果 Excel    │
  └─────────────────────┘
  │
  ▼
[用户] 点击 "下载结果 Excel" → 返回修改后的原文件
```

### 结果示例

```
处理前:
  package_name                | app_name
  com.tencent.ig              | PUBG
  com.tapblaze.coffeebusiness | Coffee

处理后:
  package_name                | 排查时间           | 当前后台版本号(vc) | 对比状态 | app_name
  com.tencent.ig              | 2026-06-03 18:02  | 21118             | 已匹配   | PUBG
  com.tapblaze.coffeebusiness | 2026-06-03 18:02  | 1001              | 已匹配   | Coffee
```

### 控制

```
暂停: POST /api/batch/{id}/pause  → asyncio.Event
继续: POST /api/batch/{id}/resume
取消: POST /api/batch/{id}/cancel
```

## 十、记忆化

```
输入包名 → 点击 "应用上次版本"
  GET /api/memo/{pkg} → 自动填充版本号/版本名
排查完成 (save_memo=true) → POST /api/memo → SQLite UPSERT

规则:
  ✅ 单包手动排查
  ❌ 多包名不触发
  ❌ Excel批量不触发 (避免污染)
```
